## 🏗️ Architecture Overview

### Core Concept: Dual Content System
Mindwood supports **two types of audio messages** within the same jewelry item:

1. **`murmures_IA`** (AI Messages)
   - Text generated by AI prompt
   - Converted to speech via OpenAI TTS
   - Stored in Supabase Storage as WebM

2. **`voix_enregistree`** (User-Recorded)
   - Audio recorded directly by user in app
   - Captured via browser MediaRecorder API
   - User can retry up to 5 times before locking

**Critical**: Each jewelry item (`bijou`) has ONE type, enforced by `useSetupSealGuard()`.

## 🗄️ Database Schema

### Key Tables
```sql
bijoux
├── id_bijou (UUID, PK)
├── type_bijou ('murmures_IA' | 'voix_enregistree')
├── destinataire_nom (text)
├── credits_messages (int)
├── credits_lectures (int)
└── metadata (JSONB)

voix_enregistrees (recorded audio)
├── id (UUID, PK)
├── id_bijou (FK → bijoux)
├── audio_url (text, Supabase Storage path)
├── duree_secondes (int)
├── enregistreur_nom (text, optional)
├── is_locked (boolean, true = no re-recording allowed)
└── created_at (timestamp)

recording_sessions (attempt tracking)
├── id (UUID, PK)
├── id_bijou (FK)
├── essais_restants (int, default 5)
├── locked (boolean)
└── created_at (timestamp)

recording_drafts (temporary)
├── id (UUID, PK)
├── id_bijou (FK)
├── audio_url (text, draft storage path)
├── duree_secondes (int)
└── created_at (timestamp)
```

### Storage Paths (Supabase S3)
```
tts/recordings/
├── [id_bijou]/
│   ├── voix/
│   │   └── [timestamp].webm          (final recorded audio)
│   └── drafts/
│       └── [timestamp].webm          (draft for preview)
```

## 🔄 Core Workflows

### 1. Recording Flow (User Records Audio)
```
User NFC Scan
    ↓
GET /setup/[id]/firstname (check type)
    ↓
→ type === 'voix_enregistree'? 
    ↓ YES
GET /api/record/config?id_bijou=[id]
    ↓
Render RecordClient (AudioRecorder + UI)
    ↓
User Records (MediaRecorder → Blob) → POST /api/record (isDraft: true)
    ↓
Save to recording_drafts + Supabase Storage
    ↓
Show Preview Playback
    ↓
User Validates → POST /api/record (isDraft: false)
    ↓
Save to voix_enregistrees + Lock Session
    ↓
Redirect to /listen/recorded/[id]
```

### 2. AI Message Flow (Murmures IA)
```
User Setup Wizard
    ↓
Select Theme + Subtheme + Recipient Details
    ↓
POST /api/generate-message (theme, recipient, etc.)
    ↓
AI Generates Prompt → French/English Text
    ↓
POST /api/tts (text → OpenAI TTS)
    ↓
Stream/Save to Supabase Storage
    ↓
GET /listen/[id] → Auto-play
```

### 3. Stripe Payment Flow
```
User Clicks "Recharge"
    ↓
POST /api/stripe/create-checkout-session
    ↓
Create Stripe session (10 or 20 message package)
    ↓
Redirect to Stripe Checkout
    ↓
User Pays
    ↓
Stripe Webhook → POST /api/stripe/webhook
    ↓
Update voix_enregistrees.credits_messages in DB
    ↓
Redirect to /recharge/[id]/success
```

```typescript

const mediaRecorder = new MediaRecorder(stream);
mediaRecorder.ondataavailable = (e) => audioChunks.push(e.data);
mediaRecorder.onstop = async () => {
  const blob = new Blob(audioChunks, { type: "audio/webm" });
  const base64 = await blobToBase64(blob);
  
  // Send to server

  formData.append("audio", blob);
  formData.append("id_bijou", id_bijou);
```

## ⚠️ Known Quirks & Pitfalls

### 1. Recording Locks (Immutable)
Once a user validates a recording (`isDraft: false`), the session is **permanently locked**. Users cannot re-record.
- **Why**: Preserve audio authenticity; prevent tampering
- **Check**: API validates `is_locked` before accepting new uploads

### 2. Draft Expiry
Draft recordings expire after a set duration (check `recording_drafts.created_at`).
- **Your Code**: Must handle stale URL responses in preview

### 3. Dual Mode Prevention
`useSetupSealGuard()` prevents switching between `murmures_IA` ↔ `voix_enregistree` after initial setup.
- **Trigger**: Redirects to `/setup/[id]/mode-scelle` if mismatch detected
- **Backend**: `/api/setup/[id]/firstname` always returns `type_bijou`

### 4. Service Role Key Exposure
The `SUPABASE_SERVICE_ROLE_KEY` **must never be exposed to client**.
- **Correct**: Use only in server-side API routes (`export const runtime = "nodejs"`)
- **Incorrect**: ❌ Never import in `"use client"` components

### 5. Audio Format Compatibility
WebM codec support in browsers:
- ✅ Chrome, Edge, Firefox on all platforms
- ❌ Safari (iOS/Mac) — no WebM support  
- **Workaround**: Check browser, fallback to MP4 if needed (currently not implemented)

### 6. Locale Detection is Static
Locale is detected once on page load from `window.location.hostname`.
- **Issue**: Won't update if user navigates domain (e.g., .fr → .com)
- **Workaround**: Full page reload required for locale switch

### 7. Stripe Webhook Testing
Local testing requires **Stripe CLI** forwarding.
- **Without CLI**: Webhook won't reach `localhost:3000`
- **In Prod**: Must add webhook URL to Stripe Dashboard → Webhooks

### 8. No Pagination on Lists
All data fetches are unbounded (no limit/offset).
- **Risk**: Large datasets slow down page loads
- **Future**: Implement pagination + infinite scroll if needed
